## What Is VANDI?

Live event venues coordinate their staff across five different apps — Telegram, Discord, Slack, SMS, and email. Vendors live in inboxes. Contracts live in folders. Data lives in spreadsheets. Nothing talks to anything else.

VANDI is the AI layer that connects all of it.

Six purpose-built AI agents handle operations, vendor management, compliance, communications, reporting, and risk — coordinated by a central orchestrator called Maestro. Your team keeps using the tools they already know. VANDI routes everything intelligently behind the scenes.

Built by an operator. For operators.

## The Agent Network

| Agent | Domain | What It Does |
|---|---|---|
| 🎼 **Maestro** | Orchestration | Routes all tasks, runs daily briefs, resolves conflicts |
| ⚙️ **Atlas** | Operations | Runsheets, scheduling, resource allocation, delay management |
| 🌐 **Vega** | Vendor Intel | Catalog management, confirmations, performance scoring |
| 📁 **Forge** | Compliance | Contracts, permits, artist riders, insurance, settlements |
| 📊 **Clio** | Reporting | Live event pacing, post-event debriefs, board analytics |
| 📡 **Loki** | Communications | Routes messages across Telegram, Discord, Slack, SMS, email |
| 🔥 **Pyrrhus** | Risk & Incident | Pre-event assessments, real-time monitoring, playbooks |

## Four Platform Pillars

- **Operations** — Real-time runsheet management, staff coordination, multi-channel routing
- **Vendors** — Centralized catalog, automated confirmations, performance tracking
- **Compliance** — Contract version control, permit tracking, rider review, settlement automation
- **Intelligence** — Live pacing dashboards, post-event debriefs, historical benchmarking
